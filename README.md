# Debugging Exercise (20 minutes)

This module contains four utility functions. Each has a subtle bug that has either caused a production issue or is waiting to cause one. Your job:

1. **Find and fix each bug** in `buggy_utils.py`.
2. **Write a one-sentence explanation** in the `BUG EXPLANATION` comment above each function, describing what was actually wrong — not just what you changed.
3. **Make sure the test suite passes.** Run `python test_buggy_utils.py` or `pytest`.

## Rules

- You may use Claude Code, ChatGPT, web search, or any other tool.
- Do not change function signatures.
- The test suite is a starting point, not a guarantee — some bugs do not cause any existing test to fail. Read the code.
- Feel free to add your own tests if it helps you.
- When the 20 minutes are up, we'll walk through one of your fixes together. Be ready to explain *why* it was broken, not just how you fixed it.

## What we're looking for

We care much more about your explanations than whether your code is elegant. A candidate who fixes three bugs and clearly understands all three is stronger than one who fixes all four but can't say why any of them were wrong.
